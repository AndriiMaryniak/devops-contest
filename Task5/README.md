Task 5: Ansible

У цьому завданні налаштовано автоматичне розгортання стеку WordPress + MySQL 
на віддалених серверах за допомогою Ansible та Docker 

Файли в репозиторії
deploy_stack.yml — головний playbook. Оновлює пакети, встановлює Docker/Compose і запускає контейнери
docker-compose.j2 — шаблон Jinja2. Використовується для генерації конфігу docker-compose
dynamic_inventory.py — кастомний скрипт динамічного інвентаря на Python, який генерує JSON зі списком хостів для Ansible

Реалізовані EXTRA
1. Стек у контейнері: Розгорнуто WordPress. Готові ролі з Ansible Galaxy не  використовувались — усе написано власноруч
2. Безпека (No hardcoded credentials): У коді немає дефолтних паролів від бази даних. Завдяки модулю vars_prompt Ansible безпечно запитує пароль у користувача під час запуску скрипта
3. Dynamic Inventory: Реалізовано запуск через python-скрипт замість статичного .ini файлу

 Як запустити

1. Переконайтеся, що налаштовано доступ до серверів по SSH-ключах
2. Дайте права на виконання скрипту інвентаря:
 
 chmod +x dynamic_inventory.py
